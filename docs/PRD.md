## 一、产品概述

### 1.1 产品定位

UniDoc 是一款以 **Markdown 为核心语法** 的轻量化全能办公编辑器。用单一软件、单一文档格式，完全平替传统 Office（Word / Excel / PPT）及常规 WPS 的全部日常办公功能。

### 1.2 产品核心理念

1. **Markdown 为核心**：所有内容以 Markdown 为底层表示，用户可直接阅读和编辑原始语法，也可在可视化编辑器中操作。
2. **无三件套割裂**：不再区分文档、表格、演示文件。文本、表格、分页、图表全部在同一个文档流内。
3. **不捆绑浏览器内核**：不携带 Chromium、CEF 或独立浏览器进程。调用系统原生 WebView 做渲染，应用本体保持轻量。
4. **AI 深度底层融合**：AI 不是插件，是软件唯一核心工作流。所有高级编辑、排版、计算、重构、演示优化由 AI 完成。
5. **完全开放 AI 接口**：不捆绑任何官方 AI 服务，用户可自由接入任意兼容 API / 本地模型。

### 1.3 目标用户

- 需要日常文档写作、数据整理、汇报演示的职场人士
- 需要编写论文、报告、台账的学生和研究人员
- 对隐私敏感、希望掌控自己 AI 后端的用户
- 厌倦 Office 臃肿和割裂体验的用户

## 二、需求优先级

本文档所有技术决策和设计取舍遵循以下优先级：

| 优先级 | 需求 | 说明 |
|--------|------|------|
| **P0** | **功能完整** | 必须完整平替 Word/Excel/PPT 的日常功能，不能有明显能力缺口 |
| **P1** | **跨平台一致性** | 同一文档在任何设备、任何系统上打开，视觉和交互体验高度一致 |
| **P2** | **轻量化** | 在满足 P0/P1 的前提下，尽量降低体积和内存占用 |
| **核心** | **AI 深度集成** | 贯穿全部功能的底层能力，用户可自定义 API |

## 三、核心设计原则

### 3.1 Markdown 原生原则

- 文档的权威表示是 Markdown，可视化编辑是 Markdown 的"所见即所得"视图。
- 用户可随时切换源码模式和可视化模式。
- 所有 Block 类型都有明确的 Markdown 序列化格式。

### 3.2 单文档大一统原则

- 所有内容类型（文本、表格、图片、分页、图表、注释）共存于同一文档。
- 无文件类型区分，一个 `.md` 文件通吃全部场景。

### 3.3 不捆绑浏览器内核原则

- 不携带 Chromium、CEF、Electron。
- 渲染层调用系统原生 WebView（Windows WebView2、macOS WKWebView、Linux WebKitGTK）。
- 应用本体仅包含业务代码和一致性保障层。

### 3.4 AI 原生驱动原则

- 软件的核心工作流由 AI 驱动。
- 本地仅保留基础输入、渲染、拖拽、保存、打印能力。
- AI 指令无处不在：全局指令栏、右键菜单、快捷键、自动建议。

### 3.5 无公式表格原则

- 彻底移除 Excel 式公式引擎、函数系统。
- 表格仅做数据展示与手动录入。
- 所有计算、改造、统计、分析由自然语言 AI 指令完成。
- 简单聚合（求和、平均值、计数）在本地即时完成，不等待 AI 响应。

### 3.6 用户自主 AI 接入原则

- 不捆绑任何官方 AI 服务。
- 完全开放自定义 API（BaseURL、API Key、模型名、参数）。
- 支持云端 API 和本地离线模型（Ollama 等）。
- API 密钥本地加密存储，软件不上传用户密钥和私有文档数据。

## 四、产品能力范围

### 4.1 替代 Word 全部主流能力

| 功能类别 | 具体能力 |
|---------|---------|
| **文本编辑** | 多级标题、段落排版、缩进、行距、字间距、字体样式（粗体/斜体/下划线/删除线/代码/上标/下标） |
| **段落格式** | 对齐方式、列表（有序/无序/任务列表）、引用块、代码块、分隔线 |
| **分页与页面** | 自由可视化分页、A4 纸张视图 |
| **页眉页脚** | 支持页眉页脚内容、页码自动插入、奇偶页不同 |
| **批注与注释** | 行内批注、文档注释、评论回复 |
| **图片与媒体** | 图片插入、缩放、裁剪、对齐（左/中/右/环绕）、拖拽调整 |
| **目录与结构** | 自动生成目录、目录点击跳转、全文格式统一、文档结构重构 |
| **查找替换** | 全文查找替换、正则支持 |
| **撤销重做** | 完整操作历史栈、分支撤销 |
| **导出** | Markdown / HTML / PDF / 图片 |

> 所有复杂排版、语体改写、结构优化由 AI 自然语言指令完成。

### 4.2 替代 Excel 全部日常办公能力

| 功能类别 | 具体能力 |
|---------|---------|
| **表格嵌入** | 文档内无限数量插入表格区块 |
| **数据录入** | 手动录入、复制粘贴、CSV 导入 |
| **行列操作** | 行列增删、拖拽排序、单元格合并/拆分 |
| **样式设置** | 单元格背景色、文字样式、边框、对齐 |
| **数据宽度自适应** | 列宽拖拽、自动适应内容 |
| **AI 数据处理** | 求和、统计、排序、筛选、新增维度、标注高亮、数据纠错、异常检测、生成图表数据源 |

> 无任何公式、无函数、无计算引擎。简单聚合由本地即时完成，复杂分析由 AI 完成。

### 4.3 替代 PPT 演示能力

| 功能类别 | 具体能力 |
|---------|---------|
| **幻灯片识别** | 复用文档内分页标记自动识别幻灯片边界 |
| **演示模式** | 一键进入全屏演示模式，键盘/鼠标/遥控器翻页 |
| **AI 演示优化** | 自动精简文案、优化每页结构、生成演讲话术建议 |
| **导出** | 演示版 PDF、单页图片导出 |

> 彻底无需单独创建 PPT 文件，写文档即做汇报。

### 4.4 统一附加办公能力

- 文档模板系统（内置模板 + 用户自定义）
- 最近文件管理
- 自动保存与恢复
- 多标签页/多窗口编辑
- 深色/浅色主题切换
- 缩放与全屏阅读模式

## 五、AI 深度融合规则

### 5.1 AI 中枢系统

所有 AI 行为统一调度，分为两类指令：

- **全局指令**：作用整篇文档
  - 结构重构（调整章节顺序、合并拆分段落）
  - 格式归一（统一字体、统一标题层级）
  - 目录生成
  - 全文润色（语体转换、降重、扩写、缩写）
  - 文档摘要

- **选区指令**：作用用户选中的任意区块
  - 文本润色、纠错、翻译
  - 表格数据处理、分析、可视化建议
  - 图片 OCR 与描述生成
  - 单页演示优化

### 5.2 AI 调用机制

| 调用类型 | 触发方式 | 处理方式 | 响应要求 |
|---------|---------|---------|---------|
| **全局指令** | AI 指令栏输入、快捷键 | 调用 AI API | 返回结构化 Markdown，自动替换全文 |
| **选区指令** | 右键菜单、AI 指令栏 | 调用 AI API | 返回替换内容或 JSON，自动应用 |
| **表格 AI** | 右键菜单、AI 指令栏 | 调用 AI API | 强制返回标准 JSON，自动刷新表格 |
| **简单聚合** | 右键菜单 | **本地计算** | 即时返回，0 延迟 |
| **智能建议** | 自动触发（可关闭） | 调用 AI API（去标识化） | 行内展示建议，用户一键采纳 |

### 5.3 系统提示词与输出规范

- 软件内置固定专业系统提示词，保证 AI 输出稳定结构化。
- 表格 AI 强制返回标准 JSON Schema，前端自动解析渲染。
- 文档级操作返回完整 Markdown，经 diff 算法最小化替换，保留用户未修改内容。

### 5.4 AI 缓存机制

- 相同自然语言指令 + 相同数据指纹 → 本地缓存直接返回。
- 缓存可手动清空，可设置过期时间。

## 六、用户自定义 AI API 系统

### 6.1 接口配置

用户可自主配置以下参数：

- BaseURL
- API Key
- 模型名称
- Temperature（温度）
- Max Tokens（最大输出长度）
- 上下文长度限制
- 超时时间
- 代理设置（可选）

### 6.2 兼容模型生态

- **云端 API**：OpenAI、Azure OpenAI、Anthropic Claude、Google Gemini、豆包、通义千问、文心一言、智谱 GLM 等所有兼容 OpenAI 标准接口的服务
- **本地离线模型**：Ollama、LM Studio、LocalAI 等本地部署模型

### 6.3 多配置文件切换

- 用户可保存多套 AI 配置方案。
- 支持一键切换：「办公模型」「本地模型」「高速模型」「长文本模型」。
- 切换即时生效，无需重启软件。

### 6.4 安全规则

- API 密钥使用系统密钥链（Keychain/Windows Credential Manager）或 AES-256 本地加密存储，不明文保存。
- 软件不上传用户 API 密钥。
- 软件不上传用户私有文档数据至任何官方服务器。
- AI 请求直连用户配置的 API 端点，无中间代理（除非用户主动设置）。
- 支持完全离线模式（仅使用本地模型）。

## 七、文档存储架构

### 7.1 文件格式 `.md`（Markdown）

文档直接保存为 Markdown 格式：

```
document.md            # 扩展 Markdown 主体内容
```

采用扩展 Markdown 语法来表达所有内容类型（表格、图片、分页等），保持纯文本的可读性和 Git 友好性。

### 7.2 扩展 Markdown 规则

在标准 Markdown 基础上扩展以下语法：

| 语法 | 说明 | 示例 |
|------|------|------|
| `---page---` | 分页节点 | 分页标记，演示模式下作为幻灯片边界 |
| `::table{id="t1"}` | 表格区块引用 | 内嵌表格，通过扩展 Markdown 语法表达 |
| `::image{id="i1" align="center" width="300"}` | 图片引用 | 图片通过扩展语法引用 |
| `::comment{id="c1"}` | 批注引用 | 批注内容通过扩展语法内联表达 |
| `::highlight{color="yellow"}` | 高亮标记 | 行内高亮 |

### 7.3 序列化规则

- 编辑态：可视化 Block 编辑器 ↔ 扩展 Markdown 双向实时同步。
- 存储态：直接以扩展 Markdown 格式保存。
- 导入：支持从标准 Markdown、Word（.docx）、HTML 导入，自动转换为 Block 结构。
- 导出：支持导出为标准 Markdown、HTML、PDF、图片。

## 八、技术架构

### 8.1 固定技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **应用框架** | Tauri v2 (Rust) | 窗口管理、系统菜单、文件对话框、全局快捷键、自动更新、安全存储、系统密钥链访问 |
| **前端框架** | Vue 3 + Vite + TypeScript | 编辑器 UI、状态管理（Pinia）、路由、组件系统 |
| **UI 组件库** | 自研 + Radix Vue / Headless UI | 轻量、可定制、无障碍支持 |
| **文档引擎** | 自研 Block-based 渲染系统 | 基于 HTML/CSS 的块级内容渲染，每个 Block 独立管理状态和渲染 |
| **表格组件** | 自研轻量 Table Block | 纯展示与录入，无公式引擎 |
| **AI 调度层** | Rust (Tauri Commands + reqwest) | HTTP 客户端、请求代理、API Key 加密管理、多配置切换、本地缓存 |
| **存储层** | Rust + JS | `.md` 文件读写、自动保存 |
| **PDF 导出** | headless 渲染或 print-to-pdf | 统一后端导出，确保跨平台输出一致 |


## 九、跨平台一致性保障规范

为确保同一文档在不同设备、不同系统上呈现高度一致的视觉效果，制定以下保障规范。

### 9.1 字体策略

- **强制打包字体**：软件安装包内嵌 Noto Sans CJK（中文）、Noto Sans（英文/数字）、JetBrains Mono（代码）。
- **禁用系统字体回退**：CSS 中 font-family 严格指定打包字体，不使用 sans-serif 等泛化回退。
- **字体加载优先级**：优先从安装目录加载，加载失败时显示占位提示（而非使用系统字体）。

### 9.2 CSS 子集规范

- **允许使用的 CSS 特性**：
  - CSS 2.1 全部特性
  - CSS Flexbox（完整）
  - CSS Grid（完整）
  - CSS 变量（Custom Properties）
  - CSS transform / opacity / box-shadow
  - CSS @media print（分页控制）
- **禁止使用的 CSS 特性**：
  - CSS Houdini
  - 实验性特性（带前缀的 -webkit-xxx 实验属性）
  - CSS Subgrid
  - 容器查询（Container Queries，WebView 支持度不一致）

### 9.3 WebView 版本要求

| 平台 | 最低要求 | 检测与处理 |
|------|---------|-----------|
| **Windows** | WebView2 Runtime >= 109 | 安装时检测，未安装引导下载；运行时检测版本过低提示更新 |
| **macOS** | >= 11.0 (Big Sur) | 系统自带 WKWebView，无需额外安装 |
| **Linux** | WebKitGTK >= 4.1 | 安装包依赖声明，缺失时提示安装 |

### 9.4 渲染一致性测试矩阵

以下场景必须在 Windows / macOS / Linux 三平台逐一验证：

- [ ] 文本排版（不同字号、行距、缩进）
- [ ] 表格渲染（边框、合并单元格、对齐、背景色）
- [ ] 图片插入（缩放、对齐、环绕）
- [ ] 分页标记（编辑态虚线、打印态分页、演示态边界）
- [ ] 页眉页脚（位置、页码、多页连续）
- [ ] 打印预览与 PDF 导出（A4 尺寸、边距、分页）
- [ ] 深色/浅色主题（全部组件颜色正确）
- [ ] 中文排版（标点挤压、行首行尾禁则）

### 9.5 度量单位规范

- 所有排版尺寸使用 `pt`（点）作为基准单位，前端按 1pt = 1.333px（96 DPI）换算。
- 禁止直接使用 `px` 设置与打印相关的尺寸（页边距、页眉页脚高度等）。
- 确保打印输出和屏幕显示的 A4 页面尺寸严格一致（210mm x 297mm = 595pt x 842pt）。

## 十、UI 交互规范

### 10.1 布局结构

```
┌─────────────────────────────────────┐
│  标题栏 (文档名称 / 软件标识)          │
├─────────────────────────────────────┤
│  工具栏 (常用格式 / 插入 / AI 入口)    │
├─────────────────────────────────────┤
│                                     │
│  编辑器区域 (Block-based 画布)        │
│  ┌─────────────────────────────┐   │
│  │  A4 纸张视图 (可切换无限滚动)  │   │
│  │  ├── Block: Heading          │   │
│  │  ├── Block: Paragraph        │   │
│  │  ├── Block: Table            │   │
│  │  ├── Block: PageBreak        │   │
│  │  └── ...                     │   │
│  └─────────────────────────────┘   │
│                                     │
├─────────────────────────────────────┤
│  AI 指令输入栏 (全局随时调用)          │
├─────────────────────────────────────┤
│  状态栏 (页码/字数/AI模型/缩放/主题)   │
└─────────────────────────────────────┘
```

### 10.2 AI 指令栏

- 位置：编辑器底部固定区域，可收起。
- 唤起方式：快捷键 `Ctrl/Cmd + K`、工具栏按钮、右键菜单。
- 交互：
  - 输入自然语言指令，按 `Enter` 执行。
  - 显示最近使用过的指令（可点击复用）。
  - 支持 `@` 提及选中的 Block。
  - 执行时显示进度指示器，完成后显示结果摘要。

### 10.3 右键菜单

选中文本/表格/图片后，右键菜单包含：

- **文本选区**：润色、扩写、缩写、翻译、纠错、语体转换、转为标题/列表/引用
- **表格选区**：求和、平均值、排序、筛选、数据分析、生成图表、高亮异常值
- **图片选区**：OCR 识别、调整大小、替换、裁剪
- **页面选区**：插入分页、转为演示页、AI 优化本页

## 十一、Block-based 文档引擎

### 11.1 核心概念

文档由一系列 **Block（区块）** 组成。每个 Block 是一个独立的内容单元，拥有唯一的 ID、类型、内容和元数据。

```
Document
├── Block #1:  heading    (level: 1, text: "项目汇报")
├── Block #2:  paragraph  (text: "本季度业绩...")
├── Block #3:  table      (id: "table-1", rows: 10, cols: 4)
├── Block #4:  page_break
├── Block #5:  heading    (level: 2, text: "财务数据")
├── Block #6:  image      (id: "img-1", width: 400, align: "center")
└── ...
```

### 11.2 Block 类型定义

| Block 类型 | 标识 | 内容结构 | 可包含子元素 |
|-----------|------|---------|-------------|
| `heading` | 标题 | 纯文本 + 行内样式 | 无 |
| `paragraph` | 段落 | 纯文本 + 行内样式 + 行内图片 | 无 |
| `table` | 表格 | 二维数据（单元格内容 + 样式） | 无 |
| `image` | 图片 | 资源引用 + 尺寸/对齐/标题 | 无 |
| `page_break` | 分页标记 | 无内容 | 无 |
| `quote` | 引用块 | 段落内容 | 无 |
| `divider` | 分隔线 | 无内容 | 无 |
| `list` | 列表 | 列表项数组（每项含文本 + 勾选状态） | 无 |
| `code_block` | 代码块 | 代码文本 + 语言标识 | 无 |

### 11.3 Block 元数据结构

每个 Block 统一包含以下元数据：

```json
{
  "id": "uuid-string",
  "type": "paragraph",
  "content": {
    // 类型特定内容
  },
  "props": {
    // 类型特定属性
  },
  "ai_history": [
    // 该 Block 的 AI 操作记录（可撤销）
  ],
  "created_at": "2026-01-01T00:00:00Z",
  "updated_at": "2026-01-01T00:00:00Z"
}
```

### 11.4 Markdown 序列化

每个 Block 类型对应明确的 Markdown 语法：

| Block | Markdown 序列化 | 反序列化规则 |
|-------|----------------|-------------|
| `heading` | `# 标题文本` | 1-6 个 `#` 对应 level 1-6 |
| `paragraph` | 普通文本段落 | 空行分隔 |
| `table` | `::table{id="xxx"}` | 嵌入表格，通过扩展语法内联存储 |
| `image` | `::image{id="xxx"}` + `![alt](path)`（可选） | 通过扩展语法引用 |
| `page_break` | `---page---` | 独立语法 |
| `quote` | `> 引用内容` | `>` 前缀 |
| `divider` | `---` | 三个或更多 `-` |
| `list` | `- item` 或 `1. item` | `-` 无序，`数字.` 有序 |
| `code_block` | \`\`\`lang\n代码\n\`\`\` | 围栏代码块 |

### 11.5 编辑交互

- **光标移动**：Block 之间用 `↑` `↓` 无缝跳转。
- **选中**：支持单 Block 选中、多 Block 连续选中（Shift + 点击）、全选。
- **拖拽**：Block 可拖拽改变顺序，表格行列可拖拽调整。
- **新建**：在空白处点击自动创建 paragraph Block；按 Enter 在当前 Block 后新建同类型 Block。
- **转换**：选中的 Block 可通过 AI 指令或快捷键转换为其他类型（如 paragraph → heading）。

## 十二、演示模式

### 12.1 幻灯片边界规则

- 文档中每个 `page_break` Block 即为一个幻灯片边界。
- 两个 `page_break` 之间的所有 Block 构成一张幻灯片。
- 如果文档开头没有 `page_break`，则从文档开头到第一个 `page_break` 为第一张幻灯片。

### 12.2 演示态渲染规则

| 元素 | 编辑态 | 演示态 |
|------|--------|--------|
| `heading` (level 1) | 正常标题 | 幻灯片大标题，居中放大 |
| `heading` (level 2+) | 正常标题 | 内容小标题，加粗突出 |
| `paragraph` | 正常段落 | 自动转为 bullet points（如有多段） |
| `table` | 完整表格 | 简化展示（关键数据高亮）或隐藏 |
| `image` | 正常图片 | 居中自适应，占幻灯片主要区域 |
| `page_break` | 虚线分隔 | 翻页边界 |

### 12.3 演示控制

- **进入**：`F5` 键或工具栏按钮。
- **翻页**：`←` `→` 方向键、`PageUp` / `PageDown`、空格、点击、滚轮。
- **退出**：`ESC` 键。
- **演讲者视图**（V2 功能）：副屏显示当前页 + 下一页 + 计时器。

## 十三、打印与导出体系

### 13.1 打印

- 调用系统打印对话框（`window.print()` 封装）。
- 打印样式通过 CSS `@media print` 精确控制：
  - A4 尺寸（210mm x 297mm）
  - 页边距可设置（默认上下 25mm，左右 20mm）
  - 页眉页脚在打印时自动插入
  - `page_break` Block 强制分页
  - 表格跨页时表头自动重复

### 13.2 PDF 导出

- **方案**：前端生成打印就绪 HTML → Rust 后端调用 headless 渲染引擎 → 输出 PDF。
- **一致性保证**：PDF 导出不依赖系统打印驱动，确保跨平台输出像素级一致。
- **选项**：
  - 页面范围（全部 / 当前页 / 指定页）
  - 页面方向（纵向 / 横向）
  - 是否包含批注

### 13.3 其他导出

| 格式 | 方式 | 说明 |
|------|------|------|
| Markdown | 直接序列化 content.md | 保留全部扩展语法 |
| HTML | 完整 HTML 文档（含内联样式） | 可独立在浏览器打开 |
| 图片 | 逐页渲染为 PNG | 可设置分辨率 |

## 十四、版本定义与产品形态

UniDoc 的最终产品形态：

> 一款以 Markdown 为核心、零臃肿、零广告、不捆绑浏览器内核、AI 底层原生驱动、用户完全掌控 AI 后端、大一统单文档、无公式表格、可完全平替 Office/WPS 的新一代轻量化智能办公编辑器。

### 核心差异化总结

1. **Markdown 原生**：开放文本格式，AI 易理解，用户可掌控。
2. **无三件套割裂**：一份文件通吃所有办公场景。
3. **无复杂公式**：自然语言搞定一切数据。
4. **不捆绑浏览器内核**：系统 WebView 渲染，极致轻量。
5. **AI 是底层能力**：不是外挂，是核心工作流。
6. **用户掌控 AI 后端**：隐私与自由度拉满。
7. **跨平台高度一致**：工程化保障，任何设备体验统一。
