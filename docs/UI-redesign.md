# UniDoc UI 改造方案

> **目标**：参考 Obsidian 布局理念，重构主编辑界面布局，提升信息密度与交互效率。
> **交付对象**：UI 设计师出稿 → 评审通过后交回前端做代码适配。
> **技术约束**：Tauri v2 + Vue 3 + TypeScript，禁用 UI 组件库（全部自研），沿用现有 CSS 语义 token。

---

## 一、改造目标总览

| # | 需求 | 当前状态 | 改造方向 |
|---|------|---------|---------|
| 1 | AI 独立窗口 | AI 命令栏嵌入主区底部 | 独立浮窗/抽屉，与主界面解耦 |
| 2 | 移除右侧属性栏 | 280px 占位面板 | 删除，编辑区横向占满 |
| 3 | 大纲可交互 | 仅展示，无点击跳转 | 点击跳转 + 当前标题高亮 + 滚动同步 |
| 4 | 左侧文件列表 | 无 | 仿 Obsidian 文件浏览器，显示当前目录文件 |
| 5 | 整体仿 Obsidian | flex 单层布局 | 三栏式 + Ribbon + 多面板 |

---

## 二、现状分析

### 2.1 当前布局结构

```
editor-layout (flex row, 100vh)
├── AppSidebar          [240px, column]    ← Logo + 导航 + 大纲(静态) + 底部按钮
└── main-area (flex column, flex:1)
    ├── TitleBar        [40px, row]        ← 文档标题 + 保存状态
    ├── Toolbar         [auto, row]        ← 全部工具按钮挤一行
    ├── editor-area (flex row, flex:1)
    │   ├── BlockEditor [flex:1]           ← A4 纸张居中
    │   └── right-panel  [280px]           ← 占位属性栏(M2 待完善) ← 待删除
    ├── AiCommandBar    [auto]             ← 底部胶囊输入 ← 待移出
    └── StatusBar       [28px]             ← 页码/字数/模型/缩放/主题
```

### 2.2 核心问题

1. **侧栏职责过载**：Logo、导航、大纲、主题切换全挤在 240px 宽的单一栏内，信息层级混乱
2. **无文件管理能力**：导航项"文档/模板/最近"全为静态占位，无法管理多文件
3. **右侧栏空置浪费**：280px 占位面板无实际功能，挤压编辑区
4. **AI 入口割裂**：AI 命令栏固定底部，与编辑上下文脱节，无法查看历史/多轮对话
5. **大纲无交互**：仅展示标题列表，无法跳转、无当前位置反馈
6. **工具栏过密**：所有操作挤一行，加粗/斜体/标题/插入/撤销/视图切换混在一起

## 三、目标布局设计（仿 Obsidian）

### 3.1 整体布局示意

```
┌─────────────────────────────────────────────────────────────────────────┐
│ TitleBar (32px)  [窗口控制] [文档路径面包屑]                [保存状态]   │
├──────┬───────────────────┬──────────────────────────────┬───────────────┤
│Ribbon│  FileExplorer     │  Editor (多标签)              │  Outline      │
│(48px)│  (220px)          │                              │  Panel        │
│      │                   │  ┌────────────────────────┐  │  (240px)      │
│ 文档 │  📁 vault-root    │  │  Tab1  Tab2  +          │  │              │
│ 搜索 │   ├─ note1.md     │  ├────────────────────────┤  │  📑 大纲      │
│ 大纲 │   ├─ folder/      │  │                        │  │   • 一级标题  │
│ AI   │   │  ├─ a.md      │  │   A4 编辑区             │  │     • 二级   │
│ 设置 │   │  └─ b.md      │  │                        │  │   • 一级标题  │
│      │   └─ note2.md     │  │                        │  │              │
│      │                   │  └────────────────────────┘  │  🏷 标签      │
│      │                   │                              │   #tag1       │
├──────┴───────────────────┴──────────────────────────────┴───────────────┤
│ StatusBar (24px) [字数] [区块数] [模型]            [缩放] [主题] [同步]  │
└─────────────────────────────────────────────────────────────────────────┘

                    + AI 独立浮窗（可拖拽/最小化，从 Ribbon 唤起）
```

### 3.2 各区域详细设计

#### A. Ribbon（最左侧图标栏，48px 宽）

**职责**：核心功能入口切换，类似 Obsidian 左侧竖条。

| 位置 | 图标 | 功能 | 交互 |
|------|------|------|------|
| 顶部 | 文档图标 | 切换 FileExplorer 面板显隐 | 单击 toggle，激活态高亮 |
| | 搜索图标 | 唤起全局搜索（Ctrl+P） | 单击打开命令面板 |
| | 大纲图标 | 切换右侧 Outline 面板显隐 | 单击 toggle |
| 底部 | AI 图标（Sparkles） | 唤起 AI 独立浮窗 | 单击打开/聚焦 AI 窗口 |
| | 主题图标 | 切换深浅色 | 单击 toggle |
| | 设置图标 | 跳转设置页 | 单击路由跳转 |

**视觉规范**：
- 宽 48px，整列垂直居中分布（顶部功能 + 底部系统）
- 图标 20×20px，行高 40px
- 默认色 `var(--muted-foreground)`，hover `var(--foreground)`，active `var(--brand-500)` + 左侧 2px 高亮条
- 背景 `var(--sidebar)`，右边框 `1px solid var(--sidebar-border)`

#### B. FileExplorer（文件浏览器，220px 宽）

**职责**：仿 Obsidian Vault 文件树，显示当前打开文件所在目录的所有 `.md` / `.uni-doc` 文件。

**结构**：
```
┌─ FileExplorer ────────────┐
│ 📁 vault-root (当前目录)   │  ← 根目录显示当前文件所在文件夹
│  ├─ 📄 欢迎文档.md         │  ← 文件图标
│  ├─ 📄 会议记录.md         │
│  ├─ 📁 笔记/               │  ← 文件夹可展开/折叠
│  │   ├─ 📄 数学笔记.md     │
│  │   └─ 📄 物理笔记.md     │
│  ├─ 📄 草稿.md             │
│  └─ 📄 模板.md             │
└───────────────────────────┘
```

**交互规范**：
- 单击文件 → 切换当前编辑文档（若未保存弹确认）
- 双击文件夹 → 展开/折叠
- 右键文件 → 上下文菜单（重命名/复制路径/删除/在文件管理器中显示）
- 当前打开文件高亮（背景 `var(--sidebar-accent)`，左侧 2px `var(--brand-500)` 条）
- 顶部 32px 标题栏：显示 vault 名称 + 右侧"新建文件"+"新建文件夹"图标按钮
- 树形缩进：每级 16px，文件夹/文件图标 16×16px

**技术约束**：
- 数据源：Tauri 文件系统 API（`@tauri-apps/plugin-fs`）扫描当前文件所在目录
- 仅显示 `.md` / `.markdown` / `.uni-doc` / `.txt` 后缀
- 排序：文件夹优先 → 字母升序
- 性能：虚拟滚动（文件数 > 200 时）

#### C. Editor（编辑区，多标签 + A4 画布）

**职责**：核心编辑区域，支持多文件标签页。

**结构**：
```
┌─ Editor ─────────────────────────────────────┐
│ [欢迎文档.md ×] [会议记录.md ×] [+]           │  ← 标签栏 (32px)
├──────────────────────────────────────────────┤
│  ┌─ Toolbar (精简) ─────────────────────┐    │  ← 浮动工具栏
│  │ B I U S | H1 H2 | • 1. ☐ | </> ⧉ ─  │    │
│  └───────────────────────────────────────┘   │
│                                              │
│        ┌─────────────────────────┐           │
│        │                         │           │
│        │      A4 编辑画布         │           │
│        │      (794 × 1123)        │           │
│        │                         │           │
│        │                         │           │
│        └─────────────────────────┘           │
│                                              │
└──────────────────────────────────────────────┘
```

**改动点**：
1. **新增多标签栏**：每打开一个文件一个 tab，支持关闭（×）/拖拽排序/右键菜单
2. **工具栏精简**：从一行 60+ 按钮改为浮动分组（仅显示常用 6-8 个 + 更多下拉）
3. **编辑区横向占满**：移除右侧 280px 属性栏，A4 画布居中显示
4. **工具栏粘性**：滚动时工具栏吸顶

**Toolbar 分组规范**：
```
[字格] B I U S | H1 H2 H3 | • 1. ☐ | </> ⧉ ─ | ↶ ↷ | ⚙
 └─格式─┘     └─标题─┘    └─列表─┘  └─插入─┘ └撤销┘ └视图┘
```
- 分组间用 1px `var(--border)` 竖线分隔
- hover 显示 tooltip（功能名 + 快捷键）
- 不常用的"插入图片/表格/代码块/分隔线/分页符"收入"⧉ 插入"下拉菜单

#### D. Outline Panel（右侧大纲面板，240px 宽）

**职责**：可交互大纲 + 标签/元信息。

**结构**：
```
┌─ Outline Panel ────────────┐
│ [📑 大纲] [🏷 标签] [ℹ 信息]│  ← Tab 切换 (32px)
├────────────────────────────┤
│ 📑 大纲                     │
│   • 一、项目概述             │  ← H1
│     • 1.1 背景              │  ← H2 (缩进 16px)
│     • 1.2 目标              │  ← H2
│   • 二、技术方案             │  ← H1
│     • 2.1 架构              │  ← H2
│       • 2.1.1 前端           │  ← H3 (缩进 32px)
│   • 三、实施计划             │  ← H1
│                            │
│ ────────────────────────── │
│ 🏷 标签 (Tab)               │
│   #数学 #物理 #笔记          │
│                            │
│ ℹ 信息 (Tab)                │
│   字数: 1,234               │
│   区块: 28                  │
│   创建: 2026-06-27          │
│   修改: 2026-06-27 19:38    │
└────────────────────────────┘
```

**大纲交互规范**（核心改造点）：
- **单击标题** → 平滑滚动到对应 block（scrollIntoView behavior:smooth）
- **当前可见标题高亮**：监听编辑区滚动，IntersectionObserver 检测当前可视 block，对应大纲项加 `var(--brand-500)` 文字色 + 左侧 2px 高亮条 + 半透明 `var(--sidebar-accent)` 背景
- **hover** → 显示 block 类型图标 + "编辑"小按钮
- **右键** → "重命名 / 折叠子项 / 复制链接"
- **拖拽**：支持拖拽大纲项调整 block 顺序（M2 可选）

**层级缩进**：`paddingLeft = 12 + (level - 1) * 16` px，level 1-6

#### E. AI 独立浮窗

**职责**：AI 对话与指令操作，独立窗口，不占主布局空间。

**形态**：可拖拽浮窗，默认从右侧滑出，支持最小化。

```
┌─ AI Assistant ──────────────────┐
│ ✨ AI 助手          ─ □ ×       │  ← 标题栏 (可拖拽)
├─────────────────────────────────┤
│ [对话] [选区操作] [全文操作]     │  ← 模式 Tab
├─────────────────────────────────┤
│                                 │
│  ┌─ 用户 ─────────────────┐    │  ← 对话历史
│  │ 帮我润色第一段          │    │
│  └─────────────────────────┘   │
│  ┌─ AI ───────────────────┐    │
│  │ 好的,这是润色后的版本... │    │
│  │ [应用到文档] [复制]     │    │
│  └─────────────────────────┘   │
│                                 │
├─────────────────────────────────┤
│ 💬 输入指令...            [→]  │  ← 输入区
│ [全文润色] [摘要] [目录] [表格] │  ← 快捷指令
└─────────────────────────────────┘
```

**交互规范**：
- **唤起方式**：Ribbon 中 AI 图标 / `Ctrl+K` / 状态栏模型名点击
- **默认位置**：屏幕右下角，距右边缘 16px，距底部 32px
- **尺寸**：宽 380px，高 540px，最小可缩到 300×400
- **可拖拽**：按住标题栏可拖到任意位置
- **可缩放**：右下角拖拽调整大小
- **可最小化**：最小化后缩为右下角小气泡（仅显示 AI 图标 + 未读红点），点击恢复
- **可关闭**：关闭后仅能通过 Ribbon/Ctrl+K 重新唤起
- **模式切换**：
  - **对话**：自由聊天，支持上下文
  - **选区操作**：对当前选中文本执行（润色/扩写/缩写/翻译）
  - **全文操作**：生成目录/摘要/优化结构
- **结果应用**：AI 输出含 `[应用到文档]` 按钮，可一键替换选区或插入

**视觉规范**：
- 浮窗阴影：`var(--shadow-xl)`
- 圆角：`var(--radius)` 收紧到 `8px`（浮窗需要更锐利）
- 背景：`var(--popover)`
- 边框：`1px solid var(--border)`
- 层级：`z-index: 1000`，浮于所有内容之上
- 用户消息气泡：背景 `var(--brand-500)`，文字 `var(--primary-foreground)`
- AI 消息气泡：背景 `var(--secondary)`，文字 `var(--foreground)`

#### F. TitleBar（顶部，32px）

**精简改动**：
- 高度从 40px → 32px（更紧凑）
- 内容从"标题 + .md + 保存圆点"改为：**窗口控制按钮 + 面包屑路径 + 保存状态**
- 面包屑：`vault-root / 笔记 / 数学笔记.md`，每级可点击
- 文档标题编辑移到编辑区顶部（A4 画布第一行 H1）

#### G. StatusBar（底部，24px）

**精简改动**：
- 高度从 28px → 24px
- 移除"AI 模型状态"（已移到 AI 浮窗）
- 保留：字数 / 区块数 / 缩放 / 主题 / 同步状态
- 点击"模型名"改为唤起 AI 浮窗

## 四、布局尺寸规范（需 token 化）

> 当前所有尺寸硬编码在各组件，改造时**必须**抽成 CSS 变量统一定义。

```css
:root {
  /* 布局尺寸 */
  --ribbon-width: 48px;
  --file-explorer-width: 220px;
  --outline-panel-width: 240px;
  --titlebar-height: 32px;
  --statusbar-height: 24px;
  --tabbar-height: 32px;
  --toolbar-height: 40px;

  /* 间距 */
  --layout-gap: 0px;          /* 各栏之间无间隙，靠 border 分隔 */
  --editor-padding: 24px;
}
```

**面板可折叠**：
- FileExplorer：`Ctrl+\` 切换显隐
- Outline：`Ctrl+Shift+\` 切换显隐
- 折叠后仅留 Ribbon，编辑区占满

## 五、交互流程改造

### 5.1 多文件管理流程

```
用户打开应用
  → FileExplorer 显示当前 vault（默认 Documents/uni-doc）
  → 单击文件 → 加载到新 Tab（若已打开则聚焦）
  → 编辑 → 自动保存（或 Ctrl+S）
  → 关闭 Tab → 若未保存弹确认
  → 新建文件 → FileExplorer 顶部"+"按钮 → 输入文件名 → 创建并打开
```

### 5.2 大纲跳转流程

```
用户在 Outline 点击 "2.1 架构"
  → scrollIntoView 到对应 heading block
  → 该 block 短暂高亮（背景闪一下 var(--brand-50)）
  → 光标聚焦到该 block 末尾
  → Outline 中该项高亮为"当前位置"
```

### 5.3 AI 工作流程

```
用户选中文本 → Ctrl+K 唤起 AI 浮窗
  → 浮窗自动切到"选区操作"模式
  → 点击"润色" → AI 返回结果
  → 点击"应用到文档" → 替换选区
  → 浮窗保留对话历史，可继续追问
```

## 七、响应式与折叠规则

### 7.1 面板折叠

| 触发 | 行为 |
|------|------|
| `Ctrl+\` | 切换 FileExplorer 显隐 |
| `Ctrl+Shift+\` | 切换 Outline 显隐 |
| Ribbon 图标点击 | 对应面板 toggle |
| 窗口宽度 < 1000px | 自动折叠 FileExplorer 和 Outline |
| 窗口宽度 < 700px | 仅显示 Editor，Ribbon 收为底部 tab bar |

### 7.2 AI 浮窗自适应

- 窗口宽度 < 800px：AI 浮窗改为底部抽屉（高度 40vh，宽度 100%）
- 窗口宽度 ≥ 800px：AI 浮窗为右下角浮窗
